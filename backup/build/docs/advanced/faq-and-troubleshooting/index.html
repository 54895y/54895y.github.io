常见问题与排错

这页只写最常见、最容易直接影响开服的问题。

1. 启动时提示 Vault 已启用，但未找到经济提供者​

典型日志：

[WARN] Vault 已启用，但未找到经济提供者

这不表示 MatrixShop 本身崩溃，而是说明：

  • 你装了 Vault
  • 但没有真正的经济插件
  • 或经济插件没有成功注册 Economy 服务

处理方式：

  • 安装实际经济插件
  • 确认经济插件已正常加载
  • 如果你不用 Vault 货币，就改用 playerpoints 或其他已配置货币

2. 打开商店没有商品​

先检查这几项：

  1. 商店 goods 里引用的商品 ID 是否真实存在
  2. 商品是否被 settings.searchable: false 或条件覆盖隐藏
  3. 商店 layout 是否真的放了商品槽位
  4. display.material 或 give.source 是否写了不存在的物品源

如果你在商店里用了内联商品：

  • 内联节点会按当前商店解析
  • 只有“引用已存在商品 ID 再覆盖字段”时，才会继承全局默认值

3. 打开商店后价格或余额显示成原样文本​

例如：

{balance}
{price}

通常是这几种原因：

  • UI 文件还是旧版本
  • 你手动复制过老示例，没有同步新的占位符
  • 菜单上下文里没有对应字段

推荐直接使用当前默认写法：

&7售价：&e{price} &7{currency_name}
&7余额：&f{balance} &7{currency_name}

4. 1.12.2 或混合核心报 NoSuchMethodError​

重点排查：

  • 是否真的替换成了最新构建产物
  • 服务端是否还在加载旧 JAR
  • 是否还有旧配置路径或旧 UI 文件

当前版本已经对这些调用做了兼容层：

  • Player.getTargetBlockExact
  • Material.matchMaterial(String, boolean)
  • 老版本耐久写法
  • 头颅 owner/profile

如果日志里仍然直接出现这些旧方法名，通常说明：

  • 服务器实际加载的不是最新 JAR

5. 箱子商店管理触发影响正常放方块​

如果你把管理触发设成了 shift_right，玩家可能会在箱子附近误触管理逻辑。
推荐改成：

settings:
  interact-trigger:
    create: shift_left
    manage: shift_left

6. 玩家商店槽位解锁没有生效​

先检查：

  • 商店模式是否为 player
  • unlock.enable 是否为 true
  • unlock.base-slots、unlock.max-slots 是否合理
  • unlock.currency 是否存在于 currency.yml
  • limit.create.limit 是否把最终槽位上限压住了

注意：

  • 最终可用槽位不是只看解锁数
  • 还会受旧的硬上限 limit.create.limit 裁剪

7. 条件价格没有命中​

排查顺序建议：

  1. 先确认 condition Kether 语句本身成立
  2. 再确认玩家上下文是否真实存在
  3. 最后检查价格模式是否写对

三种模式规则：

  • single：按顺序取第一个命中的条目
  • reduce：命中的 reduce 累加减价
  • discount：命中的 discount 连乘，支持折上折

8. 外部物品源不显示​

先检查对应前置插件是否真的加载成功，然后再确认写法。

常见写法：

material: 'source:CRAFTENGINE:namespace:item'
material: 'source:ORAXEN:item_id'
material: 'head:HDB:123'
material: 'skullsource:SKULLS:1'

如果你是通过管理命令保存主手物品：

  • 现在会先识别外部物品源
  • 再识别 mod 物品
  • 最后才回退到 repo:

9. 文档和默认配置不一致​

优先以当前插件生成的默认配置为准。
如果你从旧版本目录直接复制了文件，可能会出现：

  • 目录还是旧的短横线命名
  • 菜单文件名还是旧格式
  • 旧占位符没有同步

推荐做法：

  1. 备份旧配置
  2. 让新版本生成一套默认文件
  3. 再把旧内容按模块迁移进去

10. 首发前建议做的最小检查​

上线前至少手动测这几项：

  1. 系统商店打开与购买
  2. 玩家商店上架、购买、解锁槽位
  3. 箱子商店创建、管理、交易
  4. 玩家交易请求、取消、成交
  5. 搜索、比价、购物车、邮箱

11. 需要带什么日志来排错​

如果你要继续排错，建议至少提供：

  • 服务器核心与版本
  • Java 版本
  • 完整异常堆栈
  • 出问题时对应的配置文件
  • 你当前正在使用的 JAR 构建时间或版本号

只贴最后一行报错，通常不够定位。
