快速开始

1. 首次部署​

  1. 将 MatrixShop.jar 放入服务器 plugins/ 目录。
  2. 启动服务器，让插件生成默认配置。
  3. 关闭服务器，检查 plugins/MatrixShop/ 目录。
  4. 按你的玩法保留需要的模块目录与示例文件。
  5. 再次启动服务器，用命令验证商店是否能正常打开。

2. 默认目录​

当前版本默认会生成驼峰目录：

plugins/MatrixShop/
├─ config.yml
├─ currency.yml
├─ data.db
├─ lang/
├─ ui/
│  └─ transaction.yml
├─ systemShop/
│  ├─ module.yml
│  ├─ goods/
│  ├─ recycle/
│  ├─ shop/
│  └─ ui/
├─ playerShop/
│  ├─ module.yml
│  ├─ shop/
│  └─ ui/
├─ auction/
│  ├─ module.yml
│  ├─ shop/
│  └─ ui/
├─ globalMarket/
│  ├─ module.yml
│  ├─ shop/
│  └─ ui/
├─ chestShop/
│  ├─ module.yml
│  └─ ui/
├─ playerTrade/
│  ├─ module.yml
│  └─ ui/
├─ cart/
│  ├─ module.yml
│  └─ ui/
└─ mailbox/
   ├─ module.yml
   └─ ui/

3. 旧目录迁移​

插件启动时会尝试把旧目录迁移到新结构，例如：

  • system-shop/ -> systemShop/
  • player-shop/ -> playerShop/
  • global-market/ -> globalMarket/
  • chest-shop/ -> chestShop/
  • 平铺的 goods/、shop/、recycle/、ui/ 也会迁移

注意：

  • 迁移是“目标文件不存在时复制/移动”
  • 不会无条件覆盖你已经修改过的新文件

4. 模块开关​

主开关都在 config.yml：

model:
  system-shop:
    enable: true
  player-shop:
    enable: true
  auction:
    enable: true
  global-market:
    enable: true
  chest-shop:
    enable: true
  player-trade:
    enable: true
  cart:
    enable: true
  mailbox:
    enable: true

运行中也可以直接切换：

/msa module list
/msa module enable player-trade
/msa module disable auction

5. 打开第一个系统商店​

最简单的方式是直接使用默认示例：

/ms open example

如果你自己新建：

  1. 复制 plugins/MatrixShop/systemShop/shop/example.yml
  2. 重命名为 myShop.yml
  3. 修改 id、title、goods
  4. 保证 goods 中引用的商品已经存在于 systemShop/goods/*.yml

然后打开：

/ms open myShop

6. 最小检查清单​

在你开始正式改配置前，先确认：

  • currency.yml 里至少有一个可用货币
  • model.system-shop.enable: true
  • 商店 layout 里存在商品槽位
  • 商店 goods 中引用的商品 ID 真实存在
  • 如果你使用外部物品库，对应插件已成功加载
  • 如果你使用 Vault 货币，服务器上已经有实际经济提供者
