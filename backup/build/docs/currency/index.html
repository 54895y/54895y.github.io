货币配置

currency.yml 定义插件可用的货币。
当前实现支持三类：

  • vault
  • playerpoints
  • papi

默认示例​

currencies:
  vault:
    name: '金币'
    type: vault
    format: '${value}'

1. vault​

最常用写法：

currencies:
  vault:
    name: '金币'
    type: vault
    format: '{value} 金币'

要求：

  • 安装 Vault
  • 安装一个真正的经济提供者，例如 EssentialsX Economy、CMI 等

如果只有 Vault 没有经济插件，启动日志会提示：

Vault 已启用，但未找到经济提供者

2. playerpoints​

currencies:
  points:
    name: '点券'
    type: playerpoints
    format: '{value} 点券'

要求：

  • 服务器安装 PlayerPoints

3. papi​

这种模式适合把外部变量系统接入为货币。

currencies:
  custom_gold:
    name: '金币'
    type: papi
    papi-variable: '%custom_gold%'
    format: '{value} 金币'
    actions:
      take: 'command inline "custom take {{sender}} {amount}"'
      give: 'command inline "custom give {{sender}} {amount}"'
      deny: 'tell "&c你的金币不足。"'

说明：

  • papi-variable：读取余额
  • take：扣款动作
  • give：加款动作
  • deny：余额不足时执行

take / give / deny 都兼容 Kether 动作。

4. 当前生效字段​

每个货币节点当前真正会读取的是：

  • type
  • name
  • papi-variable
  • format
  • take
  • give
  • deny

同时兼容这些旧路径：

  • actions.take
  • actions.give
  • actions.deny

5. 菜单和占位符​

插件里会自动提供这些货币占位符：

  • {currency}
  • {currency_id}
  • {currency_type}
  • {currency_name}
  • {currency_display}

这意味着你可以在默认菜单里直接写：

- '&7售价: &f{price} {currency_name}'
- '&7余额: &f{balance} {currency_name}'

6. 一个完整案例​

currencies:
  vault:
    name: '金币'
    type: vault
    format: '{value} 金币'

  points:
    name: '点券'
    type: playerpoints
    format: '{value} 点券'

  server_token:
    name: '代币'
    type: papi
    papi-variable: '%server_token%'
    format: '{value} 代币'
    actions:
      take: 'command inline "token take {{sender}} {amount}"'
      give: 'command inline "token give {{sender}} {amount}"'
      deny: 'tell "&c你的代币不足，需要 {need}，当前只有 {balance}。"'

7. 推荐策略​

  • 普通经济服优先用 vault
  • 点券服优先用 playerpoints
  • 自定义积分系统再考虑 papi

如果你要把某个模块的可选货币限制住，不是在这里删货币，而是在对应模块的 module.yml -> economy.supported 中约束。
