# 兼容与最佳实践

这页主要回答两个问题：

- 当前版本到底兼容到什么程度
- 实际开服时怎么配更稳

## 1. 版本与核心

当前目标是：

- 运行兼容 `1.12.2+`
- 推荐测试 `1.20.4 - 1.21.x`
- 推荐核心：`Paper`、`Purpur`、`Folia`
- 已兼容常见混合核心：`Mohist`、`CatServer`、`Youer`、`Arclight`、`Magma`、`Banner`

## 2. 已做的兼容处理

当前代码里已经做了这些兼容层：

- `Material.matchMaterial` 运行时降级
- `Player.getTargetBlockExact` 运行时兼容
- 头颅 owner/profile 兼容
- 老版本耐久写法兼容
- 发包窗口兼容
- 默认打包移除 `plugin.yml` 里的 `api-version`

## 3. 开服建议

### 普通单服

- 用 `SQLite`
- 开启 `system-shop`、`cart`、`mailbox`
- 其他模块按需启用

### 大型经济服

- 考虑 `MySQL`
- 保留交易记录
- 保留性能监控
- 仔细限制可用货币与外部物品源

### 混合核心

优先做这些测试：

1. 打开系统商店
2. 购买发货
3. 箱子商店交互
4. 玩家交易请求与成交
5. 外部物品源解析

## 4. 旧配置迁移建议

- 目录统一迁移到驼峰目录
- 旧键名只当兼容层，不要继续作为新规范
- 新配置优先使用：
  - `ui.confirm.enable`
  - `file.enable_listen`
  - `settings.open-requirement`
  - `settings.search-enabled`

## 5. 推荐的改配置顺序

1. 先确认模块开关
2. 再确认货币
3. 再确认商品与商店
4. 最后改 UI

这样出问题时最容易定位。

## 6. 对生产服最重要的几条建议

- 先复制默认示例，再改，不要从空文件硬写
- 外部物品源优先直接写来源，不要先保存成 repo
- 涉及模块目录迁移时，尽量停服处理
- 箱子商店优先使用 `shift_left` 触发管理
- 玩家交易建议保留邮箱兜底
