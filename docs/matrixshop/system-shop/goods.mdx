# 系统商店商品

系统商店商品定义放在：

```text
plugins/MatrixShop/systemShop/goods/
```

每个文件下可以定义多个商品节点。

## 1. 基础示例

```yml
diamond_sword:
  buy-price: 220
  sell-price: 80
  currency: vault

  display:
    material: 'minecraft:diamond_sword'
    name: '&f钻石剑'
    lore:
      - '&7基础售价 220'

  give:
    mode: source
    source: 'minecraft:diamond_sword'
    amount: 1
```

## 2. 常用字段

### `buy-price`

支持两种写法。

#### 直接数值

```yml
buy-price: 220
```

#### 条件价格结构

```yml
buy-price:
  base: 220
  mode: discount
  entries:
    - name: '会员九折'
      condition: |-
        has permission "shop.vip"
      discount: 0.9
```

支持模式：

- `single`
- `reduce`
- `discount`

### `sell-price`

```yml
sell-price: 80
```

当前仍是普通数值，不走条件价格结构。

### `currency`

```yml
currency: vault
```

必须能在 `currency.yml` 中找到。

### `display`

```yml
display:
  material: 'minecraft:diamond_sword'
  name: '&f钻石剑'
  lore:
    - '&7这里是显示描述'
```

### `give`

```yml
give:
  mode: source
  source: 'minecraft:diamond_sword'
  amount: 1
  actions:
    - 'tell "&a你收到了钻石剑"'
```

当前支持：

- `mode`
- `source`
- `amount`
- `actions`

其中 `give.actions` 支持 Kether 动作。

### `settings`

```yml
settings:
  searchable: true
```

为 `false` 时，不参与搜索和比价。

## 3. `rules`

商品支持条件规则覆盖：

```yml
diamond_sword:
  buy-price: 220
  currency: vault
  give:
    amount: 1

  rules:
    - priority: 100
      condition: |-
        has permission "shop.vip"
      buy-price: 100
      currency: points
      give:
        amount: 2
```

说明：

- 规则按 `priority` 选中
- 同优先级按文件内顺序
- `condition` 支持单行、多行字符串和字符串列表
- 缺少某个字段时，回退默认值

## 4. 外部物品源与头颅

当前物品解析层兼容：

- 普通原版材质
- `repo:`
- `head:` / `skull:`
- `headsource:` / `skullsource:`
- `source:`
- `mod:`
- JSON 物品

### 头颅示例

```yml
display:
  material: 'head:Notch'
```

```yml
display:
  material: 'head:HDB:123'
```

```yml
display:
  material: 'skullsource:SKULLS:1'
```

### 外部物品源示例

```yml
display:
  material: 'source:CRAFTENGINE:namespace:item_id'
```

```yml
give:
  mode: source
  source: 'source:ORAXEN:item_id'
```

## 5. 商店内联商品

除了全局 `goods/*.yml`，商店文件里的 `goods:` 也支持内联商品节点。  
如果内联节点 ID 已存在于全局商品中，则商店里的写法会继承全局默认值，再覆盖你写出来的字段。

## 6. 推荐案例

```yml
vip_pickaxe:
  buy-price:
    base: 500
    mode: reduce
    entries:
      - name: 'VIP 立减'
        condition: |-
          has permission "shop.vip"
        reduce: 100
      - name: '活动再减'
        condition: |-
          papi "%server_event%" == "spring"
        reduce: 50

  sell-price: 200
  currency: vault

  display:
    material: 'source:CRAFTENGINE:tools:vip_pickaxe'
    name: '&6VIP 镐子'
    lore:
      - '&7基础售价 500'
      - '&7支持条件减价'

  give:
    mode: source
    source: 'source:CRAFTENGINE:tools:vip_pickaxe'
    amount: 1
    actions:
      - 'tell "&a物品已发放到你的背包"'

  settings:
    searchable: true
```
