# 玩家交易

玩家交易模块提供类似 TradeMe 的面对面双人交易流程。

完整流程是：

1. 发起请求
2. 对方接受
3. 双方放入物品与货币
4. 任意变更重置确认
5. 双方确认后成交

## 目录

```text
plugins/MatrixShop/playerTrade/
├─ module.yml
└─ ui/
   ├─ trade.yml
   └─ tradeCurrency.yml
```

## 模块配置示例

```yml
id: player-trade
name: 玩家交易

paths:
  ui: ui

economy:
  supported:
    - vault
    - points

settings:
  request-timeout-seconds: 30
  session-timeout-seconds: 300
  max-trade-distance: 8.0

  trigger:
    enable: true
    mode: shift_right_player

  cancel-on-close: true
  cancel-on-quit: true
  cancel-on-world-change: true
  overflow-mode: mailbox
  confirm-reset-on-change: true
```

## 1. 命令

```text
/ms trade <player>
/ms trade accept [player]
/ms trade deny [player]
/ms trade help
```

## 2. 交互触发

如果开启：

```yml
settings:
  trigger:
    enable: true
    mode: shift_right_player
```

则玩家可以通过“潜行 + 右键玩家”发起交易请求。

## 3. 交易内容

当前版本支持：

- 物品
- MatrixShop 货币

而且是一笔交易中可同时报价多种货币。  
货币编辑通过二级菜单进入，金额输入走聊天输入。

## 4. 安全规则

交易会在以下情况下自动取消：

- 任意一方关闭菜单
- 任意一方掉线
- 任意一方切换世界
- 双方距离超过 `max-trade-distance`
- 请求超时
- 会话超时

取消后，自己的报价物品会返还给自己；放不下的部分会进入邮箱。

## 5. 交易完成

成交时会执行：

1. 再次校验双方在线、世界与距离
2. 校验全部币种余额
3. 扣除全部报价货币
4. 将物品交给对方
5. 将货币打给对方
6. 记录货币流水

如果中途任一币种扣款失败，会整笔回滚。

## 6. UI 文件

- `trade.yml`：主交易界面
- `tradeCurrency.yml`：货币编辑菜单

主界面默认布局里：

- `O`：自己的报价区
- `P`：对方的报价区
- `M`：货币编辑
- `C`：确认
- `X`：取消
