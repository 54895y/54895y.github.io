# 数据库与存储

## 支持后端

`database.yml` 当前支持：

- `sqlite`
- `mysql`
- `redis`
- `yaml`

默认值：

```yml
type: sqlite
```

## 当前实现方式

当前版本采用统一快照持久化：

- 邮件记录
- 玩家个人仓库
- 待收区
- 实体仓库设置与库存快照

其中原版容器型实体仓库会优先使用方块自身的原生库存；快照层同时保留一份仓库内容备份，避免世界保存异常时完全丢失状态。

## 不自动回退

如果目标后端初始化失败，插件不会自动切到其他后端，以避免数据分裂。
