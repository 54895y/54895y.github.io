## 为什么现在插件 jar 这么小

因为当前已经改成精简发行。数据库驱动和部分第三方库不再打进主 jar，而是在首次启动时由 TabooLib 运行时依赖机制下载。

## 首次启动为什么比较慢

通常是因为正在拉取：

- Gson
- HikariCP
- SQLite JDBC
- MySQL JDBC

第一次完成后，后续启动会快很多。

## 离线服务器怎么办

有两种常见做法：

- 先在可联网环境启动一次，让依赖缓存落地。
- 手动复制对应运行时依赖缓存。

## 为什么切换账号后会被踢出

这是预期行为。MatrixAuth 需要在重新登录阶段让新的 UUID 和名称重新生效，所以切换档案或切换账号类型后会要求重新进入服务器。

## 为什么 Java 玩家能点聊天栏，基岩玩家不行

这是刻意设计。基岩玩家默认只收到纯指令提示，避免不同代理链路下点击事件表现不一致。

## QQ 群里提示找不到档案怎么办

优先检查：

- EasyBot 查询接口是否真的已开启。
- `base-url` 和 `token` 是否一致。
- QQ 是否已经绑定到 EasyBot 玩家记录。
- 该玩家是否真的在 MatrixAuth 里创建过档案。

## 为什么 `%matrixauth_profile_*%` 为空

这通常表示当前解析的玩家虽然存在基础记录，但当前身份没有命中某个具体档案。此时可以优先检查：

- `%matrixauth_has_profiles%`
- `%matrixauth_profile_count%`
- `%matrixauth_has_profile%`

## 为什么基岩玩家会只看到指令提示

这是向导系统的兼容策略。Java 版支持点击执行，基岩版只显示文本命令。
